## Tooling

Produced in **Claude Design** (Anthropic's research-preview visual prototyping product, at claude.ai) rather than an HTML artifact or a Figma file generated directly in this session — Claude Design isn't a tool available inside Claude Code, so the wireframe is generated by pasting a content/structure brief into it directly, outside this session, then reviewed together.

## Section-by-Section Rationale

1. **Nav** — logo/name + Work · About · AI in Design. Sticky vs. static left open — not a structural decision, revisit during visual design.
2. **Hero** — ~~text-forward, no imagery required~~ **superseded 2026-07-09.** Original Stage 1 premise: text-forward, no imagery, nothing competing with the headline, written for the Recruiter persona's first ~30 seconds (`content-strategy.md`). Now committed instead to **The Projection** — a physical-room opening scene with a projector casting the interface onto a wall, interactive by frame 6 (~3.4s). Full sequence, timing, visual/motion direction, and production approach: `docs/interaction-language.md` §7. The Recruiter's time budget is still the binding constraint — it's now satisfied by "interactive within ~3.5 seconds" rather than "no motion at all."
3. **Client/company logo strip** — ANZ, MYER, VicSuper, Goodwork, ACEM, Restore. Added after the initial wireframe pass to close a gap `docs/homepage-strategy.md`'s audit explicitly named: "zero social proof — no testimonials, no client logos." Deliberately quiet and compact — logo marks only, no links, no heading treatment matching the numbered sections around it — its job is a 2-second trust signal between the hero and the case studies, not content competing for attention. OUA excluded (NDA).
4. **Flagship case studies (5)** — **a tall editorial list, not a grid.** `docs/homepage-strategy.md` named the original site's flat, equal-weight grid as a specific weakness ("reads as a portfolio archive, not a curated argument for seniority"). A grid here would reproduce that problem for the 5 projects carrying the most persuasive weight. Order (from `docs/decisions.md`, a hand-curated narrative arc, not chronological): ACEM CPD Portal → Restore for Retail → Goodwork → MYER → VicSuper. Leadership/systems-thinking signal (Goodwork's design-system governance, ANZ's squad context) lives inside these case studies — it is not repeated in section 5.
5. **AI-in-Design teaser** — visually distinct band, not styled as a 6th project card. Clarified 2026-07-08 (`docs/decisions.md`): this section's job is specifically to surface the `/ai-in-design` page (currently buried in the live site's nav) — it is not a general "leadership proof point," since that job is already done by the case studies above it. Earlier drafts of this doc used the compound label "AI-in-Design / leadership proof point," which conflated the two.
6. **"More work"** — deliberately lower visual weight (smaller thumbnails, denser layout): PageUp, Restore Photo Markup, ANZ Assured, Beeline, VeroGuard, and OUA. OUA carries an NDA badge (`nda: boolean` on the project data, per `docs/decisions.md` — not a separate tier).
7. **Brand & Identity teaser** — Rebel Lemming, visually separated from "More work" since it's a different skill set (brand/identity vs. UX/product).
8. **Footer** — CTA band: email, CV download, LinkedIn.

## Constraints Carried Into This Stage

- Wireframe fidelity only — grayscale, placeholder labels.
- Mobile-first: both desktop and mobile views required, especially for how the flagship list and "More work" row reflow.
- Flagship section must read as the clearly dominant element on the page.

## Explicitly Out of Scope

- Real copy/headlines — Stage 2.
- Colors, typography, spacing for the rest of the page — Stage 3 (feeds `design-system.md`). **Exception:** the hero's own visual direction (warm/cinematic, per The Projection) is now being actively developed ahead of the rest of Stage 3 — see the staged roadmap referenced in `docs/decisions.md` (2026-07-09) — since it's load-bearing for the hero's committed motion sequence, not a later refinement of it.
- Actual Next.js implementation (`src/components/`, `src/app/page.tsx`) — after design stages are settled, per the routing/component structure already defined in `docs/information-architecture.md`. The Projection's own implementation (Next.js + Tailwind + Framer Motion) still gets its own plan-mode approval before any code is written.
